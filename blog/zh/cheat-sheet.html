流程图 (Flowchart)

flowchart TD
    %% 节点定义
    A[开始]
    B{判断}
    C[处理]
    D((结束))

    %% 连线
    A --> B
    B -->|是| C
    B -->|否| D
    C --> D

    %% 子图
    subgraph 子流程
        C --> E[步骤1]
        E --> F[步骤2]
    end
语法说明
flowchart TD从上到下 (Top Down)
flowchart LR从左到右 (Left Right)
flowchart BT从下到上 (Bottom Top)
flowchart RL从右到左 (Right Left)
A[文本]矩形节点
A(文本)圆角矩形
A((文本))圆形
A{文本}菱形（判断）
A[/文本/]平行四边形
A --> B实线箭头
A -.-> B虚线箭头
A ==> B粗线箭头
A --文本--> B带标签的箭头

时序图 (Sequence Diagram)

sequenceDiagram
    actor 用户
    participant 前端
    participant API
    participant 数据库

    用户->>前端: 点击登录
    前端->>API: POST /login
    API->>数据库: 查询用户
    数据库-->API: 返回数据
    API-->前端: 返回 token
    前端-->用户: 登录成功

    %% 循环
    loop 重试机制
        API->>数据库: 重试查询
    end

    %% 条件
    alt 用户存在
        API-->前端: 正常响应
    else 用户不存在
        API-->前端: 错误响应
    end
语法说明
participant 名称定义参与者
actor 名称定义角色
A->>B: 消息实线箭头（发送）
A-->>B: 消息虚线箭头（返回）
A->>+B: 消息激活 B
A-->>-B: 消息取消激活 B
loop 描述循环
alt 条件条件分支
opt 条件可选执行
par 并行并行执行
rect rgb(255,0,0) 背景彩色背景块

类图 (Class Diagram)

classDiagram
    class Animal {
        +String name
        +int age
        +makeSound()
        +move()
    }

    class Dog {
        +breed
        +fetch()
    }

    class Cat {
        +color
        +climb()
    }

    Animal <|-- Dog : 继承
    Animal <|-- Cat : 继承
    Dog o-- Toy : 聚合
    Cat *-- Whisker : 组合
语法说明
class 类名 { }定义类
+属性公有属性/方法
-属性私有属性/方法
#属性保护属性/方法
A <|-- B继承（B 继承 A）
A --* B组合（强依赖）
A --o B聚合（弱依赖）
A --> B关联
A ..> B依赖
A --|> B实现
A "1" --> "*" B一对多关系

状态图 (State Diagram)

stateDiagram-v2
    [*] --> 待付款
    待付款 --> 待发货: 支付成功
    待付款 --> 已取消: 取消订单
    待发货 --> 运输中: 发货
    运输中 --> 待收货: 到达
    待收货 --> 已完成: 确认收货
    已完成 --> [*]

    %% 复合状态
    state 支付流程 {
        [*] --> 选择支付方式
        选择支付方式 --> 处理中
        处理中 --> 支付成功
        处理中 --> 支付失败
    }
语法说明
[*]开始/结束状态
状态A --> 状态B状态转换
state 名称 { }复合状态
状态: 备注状态备注

ER 图 (Entity Relationship)

erDiagram
    CUSTOMER {
        string customer_id PK
        string name
        string email
    }
    ORDER {
        int order_id PK
        date order_date
        string customer_id FK
    }
    CUSTOMER ||--o{ ORDER : places
语法说明
ENTITY { }定义实体
PK主键
FK外键
||--o{一对多关系
||--||一对一关系
}|--|{多对多关系

甘特图 (Gantt Chart)

gantt
    title 项目开发计划
    dateFormat  YYYY-MM-DD

    section 设计阶段
    需求分析      :done, a1, 2026-01-01, 7d
    UI 设计       :active, a2, after a1, 5d
    原型评审      :a3, after a2, 3d

    section 开发阶段
    后端开发      :b1, after a3, 14d
    前端开发      :b2, after a3, 14d
    接口联调      :b3, after b1, 5d

    section 测试阶段
    功能测试      :c1, after b3, 7d
    Bug 修复      :c2, after c1, 5d
    上线发布      :milestone, c3, after c2, 1d
语法说明
dateFormat日期格式定义
section 名称阶段分组
:done已完成状态
:active进行中状态
:crit关键路径
:milestone里程碑
after 任务依赖前置任务
7d持续 7 天

看板 (Kanban) BETA

kanban

    [待办]
        [ ] 需求分析

    [进行中]
        [>] 后端开发
            assigned 张三

    [已完成]

[ ] 任务待办任务
[>] 任务进行中任务
[x] 任务已完成任务
assigned 姓名指派给
due 日期截止日期

雷达图 (Radar Chart) NEW v11.6

radar-beta
    title 技能评估
    axis 前端, 后端, 设计, 测试, 运维
    curve 小明{80, 70, 60, 75, 65}
    curve 小红{60, 85, 90, 70, 80}
语法说明
radar-beta声明雷达图（beta 版本）

max 数值最大值（用于缩放）
graticule circle网格样式（circle/polygon）
showLegend true显示图例

象限图 (Quadrant Chart)

quadrantChart
    title 任务优先级矩阵
    x-axis 低重要性 --> 高重要性
    y-axis 低紧急性 --> 高紧急性

    %% 象限内的点
    "修复严重bug": [0.8, 0.9]
    "发布新功能": [0.9, 0.6]
    "文档更新": [0.3, 0.4]
    "技术债务": [0.5, 0.3]
语法说明
quadrantChart声明象限图
x-axis 标签 --> 标签X 轴标签
y-axis 标签 --> 标签Y 轴标签
"名称": [x, y]数据点（坐标范围 0-1）

架构图 (Architecture) BETA

architecture-beta
    group api[API 层]

    service db(database)["MySQL"] in api
    service server(server)["Node.js"] in api

    service client(cloud)["Web 应用"]

    junction junc

    client:B --> T:server
    server:B --> T:db
语法说明
architecture-beta声明架构图（beta 版本）
service id(icon)["名称"]定义服务节点
group 名称[标签]定义分组
in 分组将服务放入分组
junction 名称连接点
A:B --> T:B从 A 底部连到 B 顶部
T:顶部 R:右 L:左 B:底连接方向

["原因1", "原因2"]中骨（具体原因）
