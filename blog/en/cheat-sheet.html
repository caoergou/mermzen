📊 Basic Diagrams (Most Used)

Flowchart

flowchart TD
    %% Node definitions
    A[Start]
    B{Decision}
    C[Process]
    D((End))

    %% Connections
    A --> B
    B -->|Yes| C
    B -->|No| D
    C --> D

    %% Subgraph
    subgraph Subprocess
        C --> E[Step 1]
        E --> F[Step 2]
    end
SyntaxDescription
flowchart TDTop-down direction
flowchart LRLeft-to-right
A[text]Rectangle node
A(text)Rounded rectangle
A((text))Circle
A{text}Diamond (decision)
A --> BSolid arrow
A -.-> BDotted arrow
A ==> BThick arrow

Sequence Diagram

sequenceDiagram
    actor User
    participant Frontend
    participant API
    participant Database

    User->>Frontend: Click login
    Frontend->>API: POST /login
    API->>Database: Query user
    Database-->API: Return data
    API-->Frontend: Return token

    loop Retry
        API->>Database: Retry query
    end

    alt User exists
        API-->Frontend: Success
    else Not found
        API-->Frontend: Error
    end
SyntaxDescription
A->>B: MessageSolid arrow (send)
A-->>B: MessageDashed arrow (return)
A->>+B: MessageActivate B
A-->>-B: MessageDeactivate B
loop DescriptionLoop
alt ConditionAlternative
opt ConditionOptional

Class Diagram

classDiagram
    class Animal {
        +String name
        +int age
        +makeSound()
    }

    class Dog {
        +breed
        +fetch()
    }

    Animal <|-- Dog : Inheritance
    Dog o-- Toy : Aggregation
SyntaxDescription
class Name { }Define class
+attributePublic
-attributePrivate
A <|-- BInheritance
A --* BComposition
A --o BAggregation

State Diagram

stateDiagram-v2
    [*] --> Pending
    Pending --> Processing: Payment
    Pending --> Cancelled: Cancel
    Processing --> Shipped: Ship
    Shipped --> [*]
SyntaxDescription
[*]Start/End state
StateA --> StateBTransition
state Name { }Composite state

ER Diagram

erDiagram
    CUSTOMER {
        string customer_id PK
        string name
    }
    ORDER {
        int order_id PK
        string customer_id FK
    }
    CUSTOMER ||--o{ ORDER : places
SyntaxDescription
ENTITY { }Define entity
PKPrimary Key
FKForeign Key
||--o{One-to-many
||--||One-to-one
}|--|{Many-to-many
📅 Project Management

Gantt Chart

gantt
    title Project Plan
    dateFormat  YYYY-MM-DD

    section Design
    Requirements      :done, a1, 2026-01-01, 7d
    UI Design         :active, a2, after a1, 5d

    section Development
    Backend           :b1, after a2, 14d
    Frontend          :b2, after a2, 14d
SyntaxDescription
dateFormatDate format
section NamePhase group
:doneCompleted
:activeActive
:critCritical path
:milestoneMilestone
